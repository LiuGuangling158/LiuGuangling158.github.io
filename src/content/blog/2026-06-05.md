## 概述

今日主要完成了三大模块重构（蛋糕搭建、蜡烛渲染、礼物选择）和 5 项 Bug 修复，涉及 10+ 个文件的增删改。核心变化：蛋糕阶段从纯 CSS 绘制改为真实 PNG 图片的拖拽叠放交互；礼物界面从 emoji 彩盒改为暗色背景 + 像素纸条 + 幸运数字机制。

## 一、蛋糕搭建阶段重构 🎂

### 1.1 改动动机

原方案使用纯 CSS（`radial-gradient`、`linear-gradient`、`conic-gradient`）绘制三层蛋糕，用户点击按钮自动堆叠。问题：
- CSS 蛋糕不够美观
- 缺乏手工搭建的参与感
- 装饰只能放在蛋糕顶部

### 1.2 新方案

使用 `cake/f0.png` ~ `cake/f3.png` 四张真实蛋糕图片，用户通过 **拖拽** 手动叠放。

| 层 | 文件 | 交互 |
|----|------|------|
| 第一层（底） | `f0.png` | 拖到托盘上 → 自动吸附 |
| 第二层 | `f1.png` | 拖到 workspace 任意位置松手 → 自由放置 |
| 第三层 | `f2.png` | 同上 |
| 顶层 | `f3.png` | 同上 |

### 1.3 拖拽实现

- **Pointer Events** 统一鼠标/触摸
- 拖拽中切 `position: fixed` 确保跨层自由移动
- 掉落命中判定：f0 检测托盘区域，f1-f3 检测 workspace 区域
- 命中 → snap/place；未命中 → 平滑动画返回原位
- 图片尺寸：`clamp(110px, 32vw, 190px)`
- 预加载四张图片避免拖拽时未渲染

### 1.4 装饰系统

- 四层全部放好后解锁装饰模式
- 选择装饰 → 点击 workspace 任意位置 → 放置（范围远大于旧版的"仅蛋糕顶部"）
- 点击已放置装饰可移除
- 坐标以 0~1 比例存储，供蜡烛阶段重建

### 1.5 文件变更

| 文件 | 操作 | 说明 |
|------|------|------|
| `js/stages/stage3-cakebuild.js` | **重写** | 完整拖拽叠蛋糕逻辑（474 行） |
| `index.html` | 修改 | 新 workspace/stack 结构，4 个图片层按钮 |
| `css/style.css` | 重写蛋糕段 | 移除 ~200 行 CSS 蛋糕样式，新增拖拽/堆放/装饰样式 |

## 二、蜡烛阶段蛋糕重建 🕯️

### 2.1 按用户搭建位置渲染

蛋糕搭建阶段将每层位置（0~1 比例坐标）存入 `AppState.cakeLayerPositions`。蜡烛阶段读取后按 **absolute 定位** 原样复现用户搭建的蛋糕。

- positioned 模式容器：`clamp(280px, 75vw, 420px) × clamp(260px, 50vh, 380px)`
- 图片放大至 `clamp(130px, 36vw, 200px)`（比搭建阶段大）
- 兜底：无位置数据时使用 flex 堆叠

### 2.2 UI 调整

| 问题 | 修复 |
|------|------|
| 提示词太靠近蛋糕 | `position: fixed; top: 6%`（原为 flex 流中紧贴蛋糕） |
| 提示词太小 | 字号从 `clamp(14px,3vw,20px)` 增到 `clamp(28px,6vw,40px)`（翻倍） |
| 倒计时被蜡烛遮挡 | 原 `z-index:15` 无定位属性无效；改为全屏 flex + `position: fixed; z-index:25` |
| 倒计时动画闪回 | 原 `transform: translate(-50%,-50%)` 与 `animation transform: scale()` 冲突；改用全屏 flex 居中，animation transform 不再冲突 |
| "21岁生日快乐" 位置不对 | 从 flex 流改为 `position: fixed; top: 72%`（蛋糕正下方） |

### 2.3 文件变更

| 文件 | 操作 | 说明 |
|------|------|------|
| `js/stages/stage4-candle.js` | 修改 | `buildCakeDisplay()` 支持 positioned/stacked 双模式 |

## 四、Bug 修复

### 4.1 移动端 2D 游戏缺少跳跃键

- **文件**：`js/stages/stage2-jumpgame.js`
- **问题**：`createMobileControls` 只创建了 ◀ ▶ 两个按钮，CSS 中 `.mc-jump` 样式已定义但 JS 未使用
- **修复**：新增跳跃按钮 `<button class="mc-btn mc-jump">跳<br>跃</button>` 并绑定 `mobileControls.jump`

### 4.2 吹气球颜色与文字不对应

- **文件**：`js/stages/stage-balloonpump.js`
- **问题**：`BALLOON_NAME_ZH[idx]` 用打乱后的索引取固定顺序的颜色名，导致文字与实际颜色不匹配
- **修复**：`PUMP_BALLOON_COLORS.indexOf(color)` 找回原始索引再取名字

### 4.3 礼物界面纸条色块闪烁

- **文件**：`index.html` + `js/stages/stage5-giftbox.js`
- **问题**：HTML 中 `class="hidden"` 无对应 CSS 规则（仅 `.stage.hidden` 和 `.el-hidden` 有效），导致纸条/按钮初始可见
- **修复**：全部改为 `el-hidden`

### 4.4 倒计时动画闪回

- **文件**：`css/style.css`
- **问题**：`.countdown-display` 用 `transform: translate(-50%, -50%)` 居中，`countdownBounce` 动画用 `transform: scale()` 覆盖前者，每帧位置跳变
- **修复**：改为全屏 flex 居中（`top:0;left:0;right:0;bottom:0;display:flex;align-items:center;justify-content:center`），animation transform 不再冲突

### 4.5 蜡烛阶段提示词/倒计时 z-index 无效

- **文件**：`css/style.css`
- **问题**：`.candle-instruction` 和 `.countdown-display` 设置了 `z-index: 15/25` 但 `position` 为 `static`（默认），z-index 不生效
- **修复**：改为 `position: fixed`

## 七、技术要点总结

1. **拖拽实现模式**：`pointerdown` 记录起始偏移 → `pointermove` 中切 `position: fixed` 跟随指针 → `pointerup` 命中判定后 snap 或返回。关键是将 listener 挂在 `document` 上防止快速移动时丢失事件。

2. **位置数据跨阶段传递**：搭建阶段将用户放置位置存储为 0~1 比例坐标，蜡烛阶段通过百分比定位重建。这种归一化坐标不受容器尺寸变化影响。

3. **CSS transform 冲突**：当元素用 `transform` 做居中定位时，任何修改 `transform` 的 `@keyframes` 动画都会覆盖定位。解决方法是改用 flex/grid 居中，让 animation 独占 `transform` 属性。

4. **z-index 的前提条件**：`z-index` 只对 `position` 为 `relative/absolute/fixed/sticky` 的元素生效。`position: static` 时 z-index 会被忽略。

5. **el-hidden vs hidden**：项目中 `.el-hidden` 有 `display: none !important` 规则，`.hidden` 仅存在于 `.stage.hidden` 选择器中。给非 stage 元素添加 `hidden` 类不会隐藏它。
