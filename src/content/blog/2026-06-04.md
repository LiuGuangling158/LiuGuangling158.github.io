## 一、新增阶段：打气筒吹气球 🎈

### 1.1 功能描述

在回忆界面（Stage 0）和 2D 散步收集游戏（Stage 2）之间插入了一个过渡阶段。用户点击打气筒，每次点击吹出一个与后续游戏颜色一一对应的彩色气球，气球飘走后提示用户进入游戏"找回来"，实现叙事上的无缝衔接。

### 1.2 完整流程

```
开场动画 → 回忆卡片 → 🆕打气筒吹气球 → 2D游戏收集气球 → 拼蛋糕 → 插蜡烛 → 选礼物 → 庆典
```

### 1.3 交互设计

| 步骤 | 描述 |
|------|------|
| 进入阶段 | 显示 CSS 绘制的粉色打气筒（手柄/筒身/底座/喷嘴），身上有 ⭐💕 装饰 |
| 点击交互 | 点击打气筒 / 点击背景 / 按空格键 → 手柄下压动画 → 喷嘴处弹出一个彩色气球 |
| 气球动画 | 先 `scale(0) → scale(1.15) → scale(1)` 弹入，再左右摇摆飘向上方渐隐消失 |
| 实时反馈 | 计数器 `3 / 12` 弹跳更新，每次显示对应颜色名（"哇！一个珊瑚红的气球飞走了~"） |
| 完成提示 | 全部吹完后显示"气球都飞走了！快把它们找回来吧！🔍"→ 按钮"出发去找气球！🔍" |
| 转入游戏 | 点击按钮 → 进入 2D 游戏，收集刚刚吹走的 12 个气球 |

### 1.4 颜色一致性

打气筒阶段的 12 个气球颜色与 `stage2-jumpgame.js` 中的 `BALLOON_UNIQUE_COLORS` 完全一致：

每次进入阶段颜色顺序随机打乱，增加可玩性。

## 二、Bug 修复

### 2.1 蜡烛阶段按钮无法点击

**问题描述**：`btn-blow`（"点击吹灭蜡烛！💨"）无法点击，点击事件被遮挡。

**根因分析**：

这是一个经典的 CSS 层叠上下文（Stacking Context）冲突问题。

- `#stage-candle` 设置了 `position: absolute`，创建了**局部层叠上下文**
- `btn-blow` 使用 `position: relative; z-index: 100`，在局部上下文内有效
- `#draggable-candle` 使用 `position: fixed; z-index: 20`，处于**根层叠上下文**
- 根据 CSS 规范，`position: fixed` 元素在根上下文中，其渲染层级高于任何局部上下文内的 `position: relative` 元素，**无论后者的 z-index 值多大**

修复前层级关系：

```
根层叠上下文
├── #draggable-candle (position: fixed, z-index: 20) ← 覆盖按钮
└── #stage-candle (position: absolute, 局部上下文)
    └── btn-blow (position: relative, z-index: 100) ← 在局部上下文中，被 fixed 元素遮挡
```

**修复方案**：

将蜡烛阶段所有按钮（`btn-blow` ×2、`btn-to-gift`）改为 `position: fixed; z-index: 500`，使按钮与 draggable candle 处于同一根层叠上下文，且 z-index 更高。

修复后层级关系：

```
根层叠上下文
├── btn-blow (position: fixed, z-index: 500) ← 最高层，可点击
├── btn-to-gift (position: fixed, z-index: 500)
└── #draggable-candle (position: fixed, z-index: 20)
```

**影响范围**：`js/stages/stage4-candle.js` 中三处按钮创建代码

### 2.2 打气筒阶段空白页

**问题描述**：回忆阶段结束后进入打气筒阶段，页面显示为空白。

**根因分析**：

`#stage-balloonpump` CSS 规则中设置了 `position: relative`，覆盖了 `.stage` 基类的 `position: absolute`。这导致：

1. 元素退出绝对定位体系，回到正常文档流
2. `height: 100%` 参照物从 `#app`（有明确 `height: 100dvh`）变为内容自身，布局塌陷
3. 与其他 `position: absolute` 的 stage 元素层级关系异常

**修复方案**：

移除 `#stage-balloonpump` 中的 `position: relative`、`overflow: hidden`、`min-height` 声明，让元素沿用 `.stage` 的 `position: absolute; top: 0; left: 0; width: 100%; height: 100%`。同时去掉 `overflow: hidden` 确保气球飞出动画不被裁剪。

---

## 三、技术要点总结

1. **CSS 层叠上下文**：`position: fixed` 始终在根上下文中渲染，其层级高于任何 `position: absolute` 父容器内的 `position: relative` 子元素。跨上下文比较 z-index 无意义。

2. **多阶段索引管理**：在 7 个阶段中插入新阶段时，需要同步更新 `stageIds` 数组、`stages` 对象、`showStage` 开关、以及所有 `registerStageCleanup` 的索引参数。本次共更新了 4 个文件的相关索引。

3. **阶段样式继承**：所有 `.stage` 共用基类样式（绝对定位、flex 布局、过渡动画），各阶段通过 ID 选择器覆盖特定属性。ID 选择器应避免修改 `position` 等核心布局属性，除非有意改变定位模型。
