# 用纯 HTML/CSS/JS，我写了一个最长的“生日快乐”

> 一个零框架、零构建、纯手写的像素风交互式 H5 生日贺卡——一段定制化前置剧情 + 8 个阶段、Canvas 2D 游戏、芯片音乐、拖拽拼蛋糕、吹蜡烛、拆礼物、烟花庆典……这是一份写给最好朋友的 21 岁生日礼物。

---

## 缘起

作为一个程序员，我一直觉得——**最好的礼物不是买来的，而是亲手打造的**。

我的朋友介纬蓉即将迎来 21 岁生日。我想送她一份特别的礼物：不是商店里的商品，而是一段可以反复体验的"旅程"——从回忆过去的美好，到一起吹气球、玩游戏、拼蛋糕、吹蜡烛、拆礼物，最后在漫天烟花中定格这个瞬间。

于是，[Birthday HAPPY](https://birthday-happy.pages.dev/) 诞生了。

## 设计理念：像素可爱的温度

### 🎮 为什么选择像素风？

像素风格有一种独特的魅力——它让人想起 Game Boy 时代的美好记忆，带有一种**手工制作的温度感**。每一个像素都是精心摆放的，就像每一段回忆都是用心珍藏的。

项目的视觉核心关键词：
- **像素可爱**（Pixel Cute）：`Press Start 2P` 复古游戏字体 + 糖果色系
- **手工温度**：纯 CSS 绘制的像素按钮、手写体般的装饰元素
- **温暖柔和**：奶油色背景、粉彩渐变、柔光粒子特效

### 🎨 色彩系统

```css
--pink: #ffb3c1;      /* 主色调：温柔粉 */
--mint: #b3ffd9;      /* 点缀：薄荷绿 */
--sky: #b3d9ff;       /* 点缀：天空蓝 */
--lavender: #d9c3ff;  /* 点缀：薰衣草紫 */
--yellow: #fff3b0;    /* 点缀：暖鹅黄 */
--cream: #fff8f0;     /* 底色：奶油白 */
--chocolate: #8b5e3c; /* 巧克力：蛋糕灵魂色 */
```

12 种气球的颜色从珊瑚红到天空蓝，贯穿整个体验——它们先被吹飞，然后要你去游戏世界里捡回来，最后环绕在蛋糕周围。**每一种颜色都是一条叙事线索。**

## 前置剧情：当代码成为"治愈魔法"

在进入正式的生日旅程之前，有一个**完全个性化定制的前置剧情**——这是整个项目中我最珍视的部分。

### 背景

朋友介纬蓉在生日前夕参加了全国大学生生命科学竞赛，网评成绩是 **88.07 分**。她对这个成绩不太满意，心情低落。于是我决定——**在生日贺卡里"改写"这个成绩**。

### 完整流程

**第一幕：温柔提问**

暖色调的覆盖层，弹出卡片式对话：

> "介纬蓉，祝你生日快乐。在今天之前你似乎有不顺心的事情，对吗？"
> 按钮只有一个字：**"对"**

点击后，文字变成：

> "我希望你现在开心起来，我打算这样做……"

**第二幕：伪造的查分页面**

一个像素级仿真的"全国大学生生命科学竞赛网络评审成绩查询"页面（[临时添加.html](临时添加.html)），包含她的真实参赛信息：

- **网评成绩：88.07**

这个页面连页脚的"竞赛委员会联系方式都做得一丝不苟——**看起来完全是真的**。

**第三幕：魔法时刻**

一个对话气泡出现在分数旁边：

> "这个网评成绩不太对吧 🤔"

紧接着，魔法粒子（✨⭐💫🌟🎀🔮🎉🪄）在分数周围炸开，成绩数字在发光动画中 **88.07 → 95.07**！

然后两个气泡接连出现：

> "哼哼，这才对嘛。😏"
> "今天是你生日，要忘记这些不开心的事情哟"

最后浮现按钮："查看完毕，进入生日庆祝"。

### 技术实现

这是一个精巧的跨文档通信设计：

- `临时添加.html` 作为独立页面，通过 `<iframe>` 嵌入 `index.html`
- 父页面通过 `postMessage` 与 iframe 通信：先请求分数元素在屏幕上的位置，再触发动画
- 分数变更动画是注入到 iframe 内部的 CSS `@keyframes`（发光 + 缩放弹跳）
- 对话气泡和魔法粒子在父页面的 `position: fixed` 层渲染，浮在 iframe 上方
- 带 8 次重试的位置请求机制，确保 iframe 渲染完毕后才开始动画序列

### 为什么要这么做？

因为这个生日礼物不是模板生成的——**它是写给一个具体的人的**。她知道那 88.07 分让我朋友失落了好几天，那我就用最程序员的方式：写一个页面，把成绩"改"成 95.07，然后告诉她——**今天你生日，忘记这些不开心的事情吧。**

## 八个阶段的"生日旅程"

整个体验被设计成 8 个连续阶段，每个阶段都有独特的交互方式：

```
前置剧情(查分改分) → 开场动画 → 回忆卡片 → 吹气球
→ 散步收集游戏 → 拼蛋糕 → 吹蜡烛 → 拆礼物 → 烟花庆典
```

### 🌟 Stage 0 — 开场

Canvas 粒子系统渲染的星空粒子（爱心、星星、光点）从底部缓缓升起。标题"祝你生日快乐"逐行淡入，伴随 `Press Start 2P` 字体的像素美感。

**技术亮点**：首帧即触发 BGM 播放（满足浏览器自动播放策略），粒子对象池复用避免 GC。

### 📸 Stage 1 — 回忆卡片

6 张回忆卡片依次展示，每张包含日期、文字和照片。点击照片可以**全屏灯箱放大**查看。最后一张卡片的标题是"此刻"，内容是"很高兴能成为你的朋友"。

**技术亮点**：点击/键盘双控、图片灯箱的 `94vw/94vh` 自适应缩放、Esc 关闭。

### 🎈 Stage 1.5 — 吹气球

CSS 纯手绘的打气筒机器！点击手柄压下，一只彩色气球从喷嘴中蹦出，带着摇摆动画飞向天空。12 只气球，12 种颜色，随机顺序。

**技术亮点**：纯 CSS 绘制的打气筒（无任何图片素材），`scale(0) → 1` 的弹性弹出动画，400ms 防连点保护。

### 🎮 Stage 2 — 散步收集游戏

一个**没有失败、没有敌人、没有惩罚**的 2D 横向卷轴收集游戏。

- Canvas 2D 渲染，800×400 逻辑分辨率
- 3000px 宽的世界，有远山视差滚动、云朵、树木、灌木
- 控制像素小人左右移动，收集散布在世界各处的 12 只气球
- 手机端自动显示虚拟摇杆

**技术亮点**：
- 纯像素绘制（`pixelDraw.js` 的 2D 数组 → Canvas `fillRect`）
- 双层视差远景（运动速度分别为 0.2× 和 0.3×）
- 气球正弦波浮动动画（振幅 6px，周期约 2.1s）
- 无 Game Over——走到终点就是胜利

### 🎂 Stage 3 — 拼蛋糕

拖拽 4 层蛋糕 PNG 图片来搭建生日蛋糕！底层 `f0` 吸附到托盘，上层 `f1-f3` 自由摆放。完成后可以点击任意位置放置 8 种装饰物（草莓、樱桃、巧克力、星星、花朵、蓝莓、爱心、糖果）。

**技术亮点**：
- Pointer Events 统一触控 + 鼠标拖拽
- 比例坐标存储（0~1），保证不同屏幕尺寸下精确重建
- 拖出边界平滑归回动画（CSS `transition`）
- 托盘区域的碰撞检测与自动吸附

### 🕯️ Stage 4 — 吹蜡烛

8 个子阶段的戏剧化仪式：

```
灯光渐暗 → 拖蜡烛到蛋糕上 → 灯光亮起，蜡烛发光
→ 气球环绕蛋糕 → 倒计时 5-4-3-2-1，许愿 → 吹灭！
```

**技术亮点**：
- 蛋糕精确重建：从比例坐标反算像素偏移 × 缩放系数
- 蜡烛拖拽使用 `position: fixed` 脱离文档流
- 烟雾粒子 CSS 动画（`scale(0)` + `translateY` + `opacity`）
- 气球环绕蛋糕的径向布局算法

### 🎁 Stage 5 — 拆礼物

3 个 CSS 手绘礼物盒，点击后盖子翻开（3D 旋转动画），露出幸运数字：
- **66** — 六六大顺
- **12** — 十二分圆满
- **88** — 八八大发

三张像素纸条揭示各自的含义，全部打开后出现完整祝福语。

### 🎆 Stage 6 — 烟花庆典

这是技术含量最高的终幕：

- **标准烟花**：从底部发射，在目标高度圆形爆炸，30-50 颗粒子
- **环状烟花**（15% 概率）：空心环 + 中心白色闪光
- **稠密烟花**（30% 概率）：额外 18 颗粒子叠加
- **文字烟花**：12 句中文祝福轮流渲染到离屏 Canvas，取像素点采样后**逐点飞到目标位置形成可读的文字**，保持约 3 秒后飘落
- **五彩纸屑**：持续飘落
- **点击交互**：每次点击在对应位置额外发射 2-10 发烟花

Canvas 使用**拖尾擦除**（`rgba(0, 5, 25, 0.18)` 半透明覆盖而非完全清除），实现流畅的运动轨迹效果。

## 几个有意思的技术细节

### 1. CSS 层级战争

`position: fixed` 元素存在于**根层叠上下文**中，天然高于所有 `position: absolute` 的阶段容器。这意味着按钮放在 fixed 层级保证永远可点击，而 Canvas 烟花设为 `pointer-events: none` 让它"看得见摸不着"，点击穿透到下方的事件处理器。

### 2. 12 色气球的叙事闭环

```
吹气球（打气筒）→ 收集游戏（跳跃跑动）→ 吹蜡烛（环绕蛋糕）→ 庆典（气球展示）
```

同一个 `BALLOON_UNIQUE_COLORS` 数组贯穿 4 个阶段，颜色本身成了叙事的载体。

### 3. 文字烟花的离屏渲染

在庆典终幕中，祝福语先被渲染到离屏 Canvas（使用宋体/SimSun），然后通过 `getImageData` 采样像素点。每个非透明像素成为一个 `ShapeParticle`：
- **飞行阶段**（~0.7s）：插值飞向目标位置
- **保持阶段**（~3s）：完全不透明，文字清晰可读
- **飘落阶段**：重力驱动，渐隐飘落

这让"生日快乐"真的在天上亮了 3 秒钟。

### 4. 蛋糕坐标的比例存储

拖拽拼蛋糕在不同屏幕尺寸下需要精确重建。我用了比例坐标：
```javascript
cakeLayerPositions: [
  { name: 'f0', x: 0.5, y: 0.8 },   // 底层在水平中央、底部
  { name: 'f2', x: 0.45, y: 0.55 },  // 偏左偏上
  // ...
]
```
重建时：`pixelX = (ratioX - 0.5) * imageScale + anchorX`

## 移动端适配的诚意

这不是一个"勉强能在手机上打开"的页面，而是**从设计之初就为移动端考虑的体验**：

- 所有字体用 `clamp()` 实现流畅缩放
- `100dvh` 处理移动浏览器地址栏的视口抖动
- 游戏虚拟摇杆在手机上自动显示
- `touch-action: manipulation` 消除双击缩放
- `env(safe-area-inset-bottom)` 适配刘海屏
- Pointer Events 统一触控和鼠标，**同一套代码两种设备**

## 写在最后

这个项目从构思到完成，大约花了 2 天时间。写代码的过程中，我反复在想：**她看到这个的时候会是什么表情？**

最让我印象深刻的部分，是那个看似不起眼的 [临时添加.html](临时添加.html)。朋友赛前被网评 88.07 分打击，心情低落了很久。这个页面看起来是一个正经的竞赛成绩查询系统，但它的真正功能是——**把 88 分变成 95 分**。动画亮起的那一刻，气泡说："今天是你生日，要忘记这些不开心的事情哟"。

这可能是程序员特有的浪漫：**用一个 HTML 文件，改写一段不愉快的记忆。**

技术上说，这是一个纯前端项目，没有任何后端，只有一个 HTML 文件和一堆 JS/CSS。但情感上说，它是我用最擅长的方式——代码——来表达"生日快乐"这四个字。

如果你也想朋友做一份特别的礼物，希望这个项目能给你一些灵感。
